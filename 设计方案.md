# RustBiliDownloader 设计方案

> **项目名称**: `RustBiliDownloader` (二进制名 `rbd`)  
> **设计原则**: 纯 Rust 优先, ffmpeg 仅作混流 fallback  
> **目标**: 复刻并超越 BBDown + Yutto 的功能, 实现现代 Rust 异步下载器

---

## 0. 设计基线 (来自 `调研报告.md` 的关键结论)

| 关键事实 | 影响设计 |
|---|---|
| BBDown 是 C# 13.8k stars, 已 archived | 协议蓝本权威, 但社区维护需依赖 fork |
| Yutto 1868 stars, **Python + Rust biliass** | 弹幕渲染 / 多 profile 认证可直接借鉴 |
| Yutto `fnval=4048` WEB 端即可拿 8K/HDR/Dolby/AV1 | APP gRPC **非必须**, 大幅降低开发成本 |
| Yutto 协程并发 video+audio | 比 BBDown 强, 应纳入 |
| Yutto FFmpegCommandBuilder 链式 | 比 BBDown 字符串拼现代, 应纳入 |
| BBDown 5-mode playurl 全实现 | 安全策略, 留作 fallback |
| Fastest 6 大缺陷 | 仅借鉴 worker pool, 其余不学 |

**`fnval=4048` 解码含义** (位掩码):
- `1` useDash, `16` useHDR, `128` use4K, `256` useDolby, `1024` use8K, `2048` useAV1
- `4048 = 2048 | 1024 | 256 | 128 | 16` → 一次请求拿到 8K+HDR+Dolby+AV1

**fnval 位掩码解码算法 (Yutto 关键发现)**:
```
fnval 位定义 (B 站协议):
  bit 0  (1)      useDash       DASH 流
  bit 4  (16)     useHDR        HDR
  bit 7  (128)    use4K         4K
  bit 8  (256)    useDolby      杜比
  bit 10 (1024)   use8K         8K
  bit 11 (2048)   useAV1        AV1
→ 4048 = 2048|1024|256|128|16 = 一次拿全部
```

**WBI 签名算法 (BBDown Parser.WbiSign 翻译)**:
```
1. GET /x/web-interface/nav → img_url, sub_url (CDN 图)
2. img_key = img_url 取 ?r=xxx 后 key
3. sub_key = sub_url 取 ?r=xxx 后 key
4. mixin_key = img_key[0..32] XOR sub_key[0..32]  (按位置 XOR)
5. 对 mixin_key 前 32 位做 64-round 重排 (固定置换表)
6. w_rid = MD5(URL_query_string + mixin_key)
→ wts 当前时间戳, w_rid 一起作为 query 参数
```

**Fetcher trait 核心签名**:
```rust
#[async_trait::async_trait]
pub trait Fetcher: Send + Sync {
    /// 抓取视频元信息
    async fn fetch(&self, ctx: &FetchContext, id: &Id) -> Result<VInfo>;
    
    /// 需要的最低登录级别
    fn login_requirement(&self) -> LoginLevel {
        LoginLevel::None
    }
    
    /// 人类可读名称 (用于诊断)
    fn name(&self) -> &'static str;
}
```

**关键算法: video + audio 并发下载 (Yutto 借鉴)**:
```rust
// Yutto 的 asyncio.gather 模式
async fn download_video_and_audio(
    video_urls: &[Url],
    audio_urls: &[Url],
    sink: &mut AsyncFileBuffer,
    progress: impl ProgressSink,
) -> Result<()> {
    let video_task = download_range(video_urls, &mut sink.video, &progress);
    let audio_task = download_range(audio_urls, &mut sink.audio, &progress);
    let progress_task = show_progress(&progress);
    tokio::join!(video_task, audio_task, progress_task);
    Ok(())
}
```

**纯 Rust vs ffmpeg 边界**:
| 场景 | 纯 Rust | ffmpeg fallback |
|---|---|---|
| DASH m4s 视频 + m4s 音频 → MP4 | ✅ mp4 crate + 自研 muxer | (不必要) |
| FLV 单段 / 多段 concat | ✅ 字节拼接 | (不必要) |
| 写入章节/封面/metadata | ✅ udta/meta box | (不必要) |
| 杜比视界 (dvh1/dvhe) | ❌ 需 NAL 级别处理 | ✅ ffmpeg 5.0+ `--enable-libdav1d` |
| 音频 codec 转换 (FLAC→AAC) | ❌ 需编码器 | ✅ ffmpeg |
| AV1 → HEVC 转码 | ❌ 需编码器 | ✅ ffmpeg |

**关键设计: `m4s/merge.rs` 伪代码**:
```
1. 解析 video.m4s:
   提取 init segment (ftyp, moov, styp)
   提取 media segments (mdat)
2. 解析 audio.m4s: 同上
3. 合并: 
   - 共享 ftyp (isom)
   - 重组 moov: 含 video track + audio track
   - 每个 track 有独立的 stbl
   - 修改 stco/stsz 偏移
4. 写 mdat: 视频 mdat + 音频 mdat 串联
5. 输出 .mp4
```

### 5.8 `rbd-job::Job` 状态机

```rust
pub struct Job {
    pub id: JobId,
    pub state: Arc<RwLock<JobState>>,
    pub vinfo: VInfo,
    pub options: DownloadOptions,
    pub output_path: PathBuf,
    pub events_tx: mpsc::Sender<JobEvent>,
}

impl Job {
    pub async fn run(self: Arc<Self>) -> Result<()> {
        self.set_state(JobState::Fetching).await;
        let tracks = self.fetch_tracks().await?;
        self.set_state(JobState::Selecting).await;
        let (video, audio) = self.select_tracks(&tracks).await?;
        self.set_state(JobState::Downloading).await;
        self.download_tracks(&video, &audio).await?;
        self.set_state(JobState::Muxing).await;
        self.mux(&video, &audio).await?;
        self.set_state(JobState::Done).await;
        Ok(())
    }
}

pub enum JobState {
    Pending,
    Fetching,
    Selecting,
    Downloading,
    Muxing,
    Finalizing,
    Done { output: PathBuf },
    Skipped { reason: String },
    Failed { error: String },
    Cancelled,
}
```

### 12.2 v1.0 范围 (IN)

**8 个核心 crate** (全部进入 v1.0):
- 
bd-foundation — 错误类型 / 日志 / 路径 / 临时目录 / 进度抽象
- 
bd-core — BilibiliId / Wbi 签名 / BV<->AV / 8 个 Extractor / 5-mode playurl
- 
bd-auth — 多 profile SESSDATA + keyring + QR 登录 (WEB+TV) + buvid 主动取
- 
bd-playurl — 4-mode 抓取 (HTML5/Web/WebBangumi/TV) + APP gRPC (备选 8K) + fnval=4048
- 
bd-danmaku — 自研 ASS 渲染 (rayon 并行) + 颜色/字号/透明度/速度
- 
bd-subtitle — 5 套 API + JSON/ASS/SRT 互转
- 
bd-downloader — 多线程 Range + 协程并发 video+audio + aria2c JSON-RPC
- 
bd-muxer — DASH copy (纯 Rust) + ffmpeg-sidecar 兜底 (DV/HDR)
- 
bd-cli — clap + dialoguer 入口

**核心能力**:
- ✅ 8K / HDR10 / 杜比视界 / 杜比全景声 / AVC/HEVC/AV1
- ✅ 单视频 / 多 P / 番剧 (ss/ep) / 课程 / 合集 / 收藏夹 / 个人空间
- ✅ WEB + TV 双端登录, 多 profile
- ✅ DASH copy 纯 Rust mux
- ✅ 杜比视界走 ffmpeg-sidecar
- ✅ 弹幕 ASS 输出 (字体/字号/速度/颜色/透明度可配)
- ✅ 多线程 Range 下载 + 协程并发
- ✅ aria2c 集成
- ✅ 交互式选 track (dialoguer)
- ✅ 配置文件 rbd.toml (TOML)
- ✅ 仅中文 (错误信息 / 日志 / 帮助)
